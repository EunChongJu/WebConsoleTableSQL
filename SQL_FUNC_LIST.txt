이거슨 에스큐엘의 사용 설명서를 요약한 것이며 표준이라 한다.

SELECT * FROM 테이블명;			: SELECT 명령 - 테이블 표시됨

세미콜론 붙이지 않고 엔터키를 누르면 입력중인 것으로 간주되어 명령문이 실행되지 않음
꼭 반드시 마지막 문장에는 세미콜론이 붙여야 된다.

SELECT 명령 구문은 '질의'나 '쿼리'라 불린다.

SELECT * FROM sample21;
------ - ---- -------- -
SELECT는 SELECT 명령을 실행하는 것, 즉 명령의 종류임
*는 모든 열을 의미하는 메타문자임.					(참고로 열은 가로의 모든 셀을 의미)
FROM은 테이블 명을 지정한다.
sample21은 테이블의 이름으로 *에 의해 모든 것을 불러온다.

다시 나누어서
SELECT * FROM sample21;
-------- --------------
SELECT 구    FROM 구
위처럼 두개의 구로 나눌 수 있다. SQL은 몇개의 구로 구성된다.

예약어는 아래와 같습니다.
SELECT * FROM sample21;
------------- ---------
  예약어임     테이블명임(예약어 아님)

데이터베이스 객체는 이름을 붙여 관리함.
같은 이름으로 다른 DB 객체를 만들 수 없음. - 이미 테이블이 정의되있다는 에러가 발생함.
SELECT라는 예약어를 테이블명으로 사용할 수 없음

DB에서 예약어와 DB 객체명은 대소문자를 구별하지 않는다.

DB에서 행과 열은 다음과 같이 설명된다.
행은 레코드라 하며 가로로 묶인 것임.
열은 칼럼 또는 필드라 하며 세로로 묶인 것임.
이처럼 행과 열이 모여 이루어진 표(테이블) 형식의 데이터임.

데이터는 자료형으로 분류할 수 있으며, 열은 하나의 자료형만 가질 수 있다.
NULL은 데이터가 들어있지 않다는 것을 의미한다. (특별한 값임)

DESC를 실행하면	- DESC sample21;	(놀랍게도 DESC는 SQL의 명령이 아님)
테이블에 어떤 열이 정의되어 있는지 알 수 있다.
+-----------+------------+------+-----+---------+-------+
| Field    | Type        | Null | Key | Default | Extra |
+----------+-------------+------+-----+---------+-------+
| no       | int(11)     | YES  |     | NULL    |       |
| name     | varchar(20) | YES  |     | NULL    |       |
| birthday | date        | YES  |     | NULL    |       |
| address  | varchar(40) | YES  |     | NULL    |       |
+-----------+------------+------+-----+---------+-------+
Field는 열의 이름이 표시됨
Type는 해당 열의 '자료형'을 나타냄
int는 Integer(정수)를 의미. 괄호 안은 최대 길이를 나타냄. 예를 들어 int(11)이면 11자리의 정수형을 저장할 수 있는 자료형임.
Null은 NULL 값을 허용할 것인지 아닌지를 나타내는 제약사항임. Yes로 하면 Null을 허용하는 거임.
Key는 해당 열이 '키'로 지정되어 있는지를 나타냄.
Default는 그 열에 주어진 '기본값' 즉, 생략했을 경우 적용되는 값임. 행 추가때 열의 데이터 값을 생략하면 이 기본값으로 채워짐.

자료형
- INTEGER	: 수치형의 하나로 정수값을 저장할 수 있는 자료형. 소수점 포함 불가.
- CHAR		: 문자열형의 하나로 문자열을 저장할 수 있는 자료형임. 열의 최대길이를 지정해야 함. 10이면 11자를 저장할 수 없음.
- VARCHAR	: CHAR와 같이 저장할 수 있음 But 데이터 크기에 맞춰 저장공간의 크기도 변경됨. 가변 길이 문자열이라 함.
- DATE		: 날짜 값을 저장할 수 있는 자료형. '1968년 6월 23일'과 같이 연월일의 데이터를 저장할 수 있음.
- TIME		: 시간을 저장할 수 있는 자료형임 '12시 30분 20초'와 같이 시분초의 데이터를 저장할 수 있는 형임.
			(제품에 따라 차이가 있다고 함)

조건을 지정하여 데이터를 검색하는 방법
SELECT 열1, 열2 FROM 테이블명 WHERE 조건식

SELECT 열1, 열2, ... FROM 테이블명에서 열을 *로 하면 전체를 반환하는 거임
SELECT * FROM sample21;에서 SELECT no, name FROM sample21;하면 no와 name의 열만 반환한다.
열을 전혀 지정하지 않은 상태에서, 존재하지 않는 열을 지정해도 실행하면 에러가 발생한다.
열 순서는 임의로 가능. 놀랍게도 열을 중복해서 지정해도 무관함.

SELECT 열 FROM 테이블명 WHERE 조건식
WHERE 뒤에는 조건식을 붙이는 거임.

SQL에서는 구의 순서가 정해져있어 바꿔적을 수 없음.
SELECT 열 WHERE 조건식 FROM 테이블명 : 이래 쓰면 에러 발생.
그러나 WHERE를 생략하면 테이블 내 모든 행이 검색대상이 되어 반환됨.

조건식은 '열과 연산자, 상수로 구성되는 식'임.
WHERE no = 2;라고 하면 no열의 2인 수를 찾아 반환하는 거임. 근데 SELECT문으로 반환됨.
즉 WHERE는 추출하는 역할이라 함.
no = 2에서
-- - -
no는 열, =는 연산자, 2는 상수이다. 연산자는 비교한 결과값이 참인지 거짓인지 반환하므로 비교연산자임.
WHERE 구로 행을 추출하지만, SELECT구로 열 지정을 동시에 할 수 있음.

비교연산자에서 <>는 서로 다른 값인지 비교하는 연산자임.

문자열은
SELECT * FROM sample21 WHERE name = '박준용';
이렇게 싱글쿼트('')로 둘러싸 표기해야 함.
날짜시간형의 경우 연월일은 하이픈(-)으로, 시각은 시분초를 콜론(:)으로 구분하여 표기한다.
문자열의 열과 비교하기 위해서는 조건으로 지정할 값도 그 자료형에 맞춰 지정해야 합니다.
한편 자료형에 맞게 표기한 상수값을 '리터럴 literal'이라 부름.
문자열 리터럴은 싱글쿼트로 둘러싸 표기한다.

수치형
1 100 -3.8

문자열형 상수
'ABC' '박준용'

날짜시간형 상수
'1989-01-07' '1991-12-26 15:21:36'

NULL 값 검색 - 하는 방법

SELECT * FROM sample21 WHERE birthday = NULL;
birthday 열 값이 NULL인 행이 있지만 birthday = NULL과 같은 조건식으로는 검색이 불가능

SELECT * FROM sample21 WHERE IS NULL;
이제 NULL 값에 해당되는 행이 표시되된다.

비교 연산자
= 연산자와 <> 연산자, IS NULL로 검색하는 방법에 관해 알아봄.
WHERE 구문으로 지정하는 조건식은 참과 거짓을 반환하는 비교 연산자나 술어를 사용해 표기할 수 있음
= 연산자 - 좌변과 우변의 값이 같을 경우 참이 된다.
<> 연산자 - 좌변과 우변의 값이 같지 않을 경우 참이 된다.
> 연산자 - 좌변의 값이 우변의 값보다 클 경우 참.
>= 연산자 - 좌변의 값이 우변의 값보가 크거나 같을 경우 참.
< 연산자 - 좌변의 값이 우변의 값보다 작을 경우 참, 같거나 클 경우 거짓.
<= 연산자 - 좌변 값 우변 값보다 작거나 같으면 참.
여기서 스페이스바 금지임.

조건 조합하기
조건식1 AND 조건식2
조건식1 OR 조건식2
NOT 조건식

실제로 SELECT 명령을 사용해서 DB에서 데이터를 검색할 때 단순한 조건식을 넣으면 많은 결과값이 나와 한눈에 알아보기 힘듬.
따라서 통상적으로 2개 이상의 조건식을 조합해 검색하는 경우가 많음.
조건식을 조합해 사용할 경우 복수의 조건을 WHERE 구로 지정.

AND로 조합하기 - 조건식1 AND 조건식2
SELECT * FROM sample21 WHERE a<>0 AND b<>0;

OR로 조합하기 - 조건식1 OR 조건식2
SELECT * FROM sample21 WHERE a<>0 OR b<>0;

- 주의사항
열이나 상수만을 지정해도 에러가 말생하지 않지만 기대한 결과값을 얻을 수 없음.
SELECT * FROM sample21 WHERE no = 1 OR 2;

올바른 사용법
SELECT * FROM sample21 WHERE no = 1 OR no = 2;

연산자 우선순위
OR보다 AND 쪽이 우선순위가 높기 때문에 AND 먼저 계산된다.

WHERE a = 1 OR (a=2 AND b=1) OR b = 2
->
SELECT * FROM sample21 WHERE (a=1 OR a=2) AND (b=1 OR b=2);

NOT으로 조합하기 - NOT 조건식
NOT 연산자는 오른쪽에만 항목을 지정하는 '단항 연산자'임.

주로 복수의 조건에 대해 '~아닌', '~외에', '~를 제외한 나머지' 등의 조건을 지정할 경우 사용함.
SELECT * FROM sample24 WHERE NOT(a<>0 OR b<>0);
